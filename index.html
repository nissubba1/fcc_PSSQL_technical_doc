PostgreSQL

PostgreSQL is an advanced, open-source object-relational database system known for its reliability, robustness, and performance. PostgreSQL supports both SQL (relational) and JSON (non-relational) querying, making it a highly versatile database system for a wide variety of data processing and analysis tasks.

It was developed as part of POSTGRES project at University of California, Berkeley. The aim behind the project was to create an advanced database technology beyond the capabilities of existing systems at the time. It was designed to support complex data types, new forms of data integrity, and allow extensibility by users.

  • ACID Compliance: Ensures that all transactions are processed reliably, maintaining data integrity.
  • Extensibility: Users can define their own data types, build out custom functions, and even write code in various programming languages without recompiling the database.
  • Advanced Data Types: Supports a wide range of data types, including geometric primitives, JSON, and XML.
  • Strong Community Support: Benefits from a vibrant, active community that contributes to its continuous development and improvement.
  • Performance and Reliability: Offers sophisticated optimization techniques, strong transactional integrity, and support for concurrent transactions, making it highly reliable and performant for enterprise applications.
  • Cost-Effective: Being open-source, it does not incur licensing costs, making it a cost-effective solution for businesses of all sizes.
Installation

In order to install PostgreSQL, make sure you have a compatible system:

  1. An operating system supported by PostgreSQL (Linux, macOS, Windows)
  2. Sufficient disk space for the database
  3. Network access for remote connectivity (optional)

Please visit official postgres website for the instruction on how to download and run on your computer.

Create Database

When you first install the PostgreSQL, you will have a database called `postgres` created for you. However, if you want to create your own database, you need to follow this format. 

          
            CREATE DATABASE database-name;
Create Table

Alright, once you have your PostgreSQL installed, and database created, it's time to start building your table. Each table will have row which requires a column name and data type. There are many different data type such as integer (int), varchar, date, and etc so before you assign a data type, make sure to look which data type is best for you. The syntax for the creating a table is: 

          
            CREATE TABLE table-name (
                  column-name data-type,
                  column-name data-type
              );

Each table need a primary key. A primary key is a unique constraint which makes sure that no duplicate value is inserted into the table. For example, primary key can be student_id, id, social_security, etc. Each table can have a foreign key. Foreign key is what links multiple table together. A foreign key must reference a primary key from another table. 

          
            CREATE TABLE table-name (
                  column-name data-type PRIMARY KEY,
                  column-name data-type
              );

OR

          
            CREATE TABLE table-name (
                  column-name data-type,
                  column-name data-type,
                  PRIMARY KEY (column-name)
              );
Insert

Once you have created the table, now its time to populate it. Table can be populated with the key word `INSERT`. When you insert a data into a table, you need to specify which table you are inserting and for what column. The syntax for the inserting into a table is: 

          
            INSERT INTO table-name 
                (column-name1, column-name2)
            VALUES 
                (column-1-value, column-2-value);

If you want to insert a multiple row at once:

          
            INSERT INTO table-name 
                (column-name1, column-name2) 
            VALUES
                (column-1-value, column-2-value), 
                (column-1-value, column-2-value),
                (column-1-value, column-2-value);
Update

The UPDATE statement is used to modify existing records in a table. It can change one or several fields at a time, and it typically includes a WHERE clause to specify which rows should be updated. If the WHERE clause is omitted, all rows in the table will be updated, which can lead to unintended data modifications. 

          
            UPDATE table-name
            SET column-name
            WHERE column-name = value;
Delete

The DELETE statement is used to remove one or more rows from a table. Similar to the UPDATE statement, it usually includes a WHERE clause to specify which rows should be deleted. If you omit the WHERE clause, all rows in the table will be deleted, which effectively empties the table but does not remove the table structure itself. 

          
            DELETE FROM table-name 
            WHERE condition;
Join

Joins in SQL are used to combine rows from two or more tables based on a related column between them. There are various types of joins, each serving different use cases.

Implicit Join

An implicit join is done using the WHERE clause to specify the condition that links the tables. 

          
            SELECT columns
            FROM table-1-name 
            , table-2-name
            WHERE table-1-name.common-field = table-2-name.common-field;
Explicit Join

An explicit join uses the JOIN keyword to specify the table to join and the ON clause to specify the condition. 

          
            SELECT columns 
            FROM table-1 
            JOIN table-2 
            ON table-1.common-field = table-2.common-field;
Left Join

The LEFT JOIN or LEFT OUTER JOIN returns all records from the left table (table1), and the matched records from the right table (table2). If there is no match, the result is NULL on the side of the right table. 

          
            SELECT columns
            FROM table-1
            LEFT JOIN table-2 
            ON table-1.common-field = table-2.common-field;
Right Join

The RIGHT JOIN or RIGHT OUTER JOIN returns all records from the right table (table2), and the matched records from the left table (table1). If there is no match, the result is NULL on the side of the left table. 

          
            SELECT columns
            FROM table-1
            RIGHT JOIN table-2 
            ON table-1.common-field = table-2.common-field;
Full Join

The FULL JOIN or FULL OUTER JOIN returns all records when there is a match in either left (table1) or right (table2) table records. It effectively combines the results of both LEFT JOIN and RIGHT JOIN. 

          
            SELECT columns
            FROM table-1
            FULL JOIN table-2 
            ON table-1.common-field = table-2.common-field;
